# Double-Entry Bookkeeping

The foundation of all accounting. Every transaction affects two accounts.

## The Accounting Equation

```
Assets = Liabilities + Equity
```

This must always balance. Every journal entry maintains this equation.

## Debits and Credits

Every transaction has two sides:

| | Debit (left) | Credit (right) |
|---|---|---|
| **Assets** | Increase | Decrease |
| **Expenses** | Increase | Decrease |
| **Liabilities** | Decrease | Increase |
| **Equity** | Decrease | Increase |
| **Revenue** | Decrease | Increase |

**Memory aid:** Assets and Expenses are debit-normal. Everything else is credit-normal.

## Journal Entry Structure

Every entry must have:
1. **Date** — when the transaction occurred
2. **Accounts** — at least two (one debit, one credit)
3. **Amounts** — debits must equal credits
4. **Description** — what happened and why

Example: Buying supplies for 1,000 + 250 VAT = 1,250 total

| Account | Debit | Credit |
|---------|-------|--------|
| Supplies (expense) | 1,000 | |
| Input VAT (asset) | 250 | |
| Bank (asset) | | 1,250 |

The entry balances: 1,250 debit = 1,250 credit.

## Why This Matters for the AI

- **Never create an unbalanced entry.** If debits don't equal credits, the entry is wrong.
- **Know which side each account type goes on.** Expenses are debited. Revenue is credited.
- **VAT is a separate line.** It's not part of the expense — it's a receivable (input) or payable (output).
- **The contra account matters.** Every expense must come from somewhere (usually bank or accounts payable).

## Common Mistakes

- Confusing debit with "money out" — debit means LEFT side, not negative
- Forgetting VAT as a separate entry line
- Creating entries with only one side (no contra account)
- Mixing up gross (incl. VAT) and net (excl. VAT) amounts
